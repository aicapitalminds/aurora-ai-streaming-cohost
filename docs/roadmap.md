# Roadmap

## Current public focus

- Document the architecture.
- Explain the build path.
- Add safe minimal examples.
- Share lessons from the private production system.

## Planned features in the private system

### Intelligent chat filtering

Aurora should not react to every chat message. Messages should be scored by:

- relevance
- intent
- novelty
- safety
- timing
- whether the streamer is busy

### Viewer memory

Remember safe, useful, stream-relevant context about recurring viewers.

### Interaction queue

Avoid interruptions by queueing possible interactions and selecting the right moment to speak.

### Better emotion/expression control

Map model output, chat tone, and gameplay events to avatar expressions/motions.

### Stream controls

Let the streamer control Aurora with hotkeys or commands:

- mute
- wake
- change mood
- clear queue
- explain last action
- switch between active/cozy modes

## Long-term vision

Aurora becomes a real character in the channel: funny, helpful, reactive, memorable, and safe.
